■ Section 3 — New Features (User-Interaction Triggered)
Each feature below is triggered by a specific user action on the website. Priority: P1 = Must-Have, P2 =
Should-Have, P3 = Nice-to-Have.
#1 Real AI Disease Detection
P1
CRITICAL
User Trigger: User clicks 'Analyze Crop' after uploading an image
Description: Send the image to a Python ML backend (FastAPI + TensorFlow/PyTorch model trained on PlantVillage
dataset). Return the top-3 disease predictions with confidence scores, severity level, and treatment steps. Replace the
current Math.random() mock completely.
Backend Needed: FastAPI inference endpoint, ML model serving (TensorFlow Serving / ONNX Runtime), image
preprocessing pipeline
#2 User Activity History / Scan Log
P1
CRITICAL
User Trigger: User opens their profile or dashboard after logging in
Description: Record every crop scan (timestamp, image thumbnail, detected disease, confidence) in a per-user
history table. Display a scrollable log so farmers can track disease trends over time on their farm.
Backend Needed: Database table: scan_history(user_id, timestamp, image_url, disease, confidence); REST API
GET /history; cloud image storage (S3/Firebase)
#3 Email / SMS OTP Verification on Registration
P1
CRITICAL
User Trigger: User submits the Register form
Description: After form submission, trigger a real OTP email (SendGrid/Mailgun) or SMS (Twilio). Activate the
account only after successful OTP entry. This replaces the completely insecure client-side verification currently in
place.
Backend Needed: Email API (SendGrid/Mailgun) or SMS API (Twilio); OTP generation & expiry stored server-side
(Redis TTL); /verify-otp endpoint
#4 Live Weather Integration
P2 HIGH
User Trigger: User lands on the home page or the 'Weather Based Tips' feature card
Description: Detect or ask for the user's location, call OpenWeatherMap API, and display current temperature,
humidity, rainfall forecast alongside contextualised crop advisory (e.g., 'High humidity detected — watch for fungal
diseases in your Kharif crops').
Backend Needed: OpenWeatherMap API integration, geolocation permission handling, server-side proxy to protect
API key
#5 Crop-Specific Disease Encyclopedia
P2 HIGH
User Trigger: User clicks on a disease card (Leaf Blight, Powdery Mildew, Rust)
Description: Open a detailed modal/page with: full disease lifecycle, affected crops, high-resolution symptom images,
organic & chemical treatment options, prevention checklist, and links to government advisories.
Backend Needed: Static JSON data store or CMS (Contentful / Strapi); image CDN for disease photos; optional
full-text search
#6 Farmer Community Forum / Q&A;
P2 HIGH
User Trigger: User clicks a new 'Community' nav link
Description: A threaded Q&A; board where registered farmers post crop problems, share photos, and receive peer /
expert answers. Include upvoting, tagging by crop type, and expert-verified answer badges.
Backend Needed: Node.js / Django backend with WebSocket support; PostgreSQL for posts/comments; image
upload to S3; moderation API
#7 Multilingual Voice Assistant
P2 HIGH
User Trigger: User clicks a microphone icon in the nav or on the hero section
Description: Allow farmers to ask questions in Hindi, Marathi, Telugu etc. via voice. Use Web Speech API (browser)
for input, send text to Claude/GPT API, and read the answer back using SpeechSynthesis. Bridges the literacy gap for
farmers who cannot type.
Backend Needed: Anthropic / OpenAI chat API; Web Speech API; server-side prompt template per crop context
#8 Soil Health Input & Fertiliser Calculator
P2 MEDIUM
User Trigger: User fills in NPK soil test values in a new 'Soil' section
Description: Accept soil NPK readings (from kit or lab report), crop type and growth stage. Compute deficiency and
display a personalised fertiliser prescription with product names, quantities, and application schedule.
Backend Needed: Rule-based calculation engine (Python); crop-nutrient requirement database; optional integration
with soil lab APIs
#9 Marketplace for Agri-Inputs
P3 LOW
User Trigger: User views a treatment recommendation after crop scan
Description: Surface recommended pesticides/fertilisers as purchasable products (affiliate or direct). Show price,
availability, and nearest retailer on a map.
Backend Needed: Product catalogue database; geolocation + Maps API; payment gateway (Razorpay); vendor
dashboard
#10 Crop Growth Timeline & Reminder
P3 LOW
User Trigger: User selects their crop and sowing date in a new 'My Farm' section
Description: Generate a personalised calendar showing key growth milestones, irrigation schedule, fertilisation
windows, and harvest estimate. Send push notifications (Firebase FCM) or WhatsApp reminders at critical times.
Backend Needed: FCM push notifications; WhatsApp Business API (Twilio); PostgreSQL schedule table; cron jobs
#11 Offline / PWA Mode
P3 LOW
User Trigger: User opens the app in a low-connectivity area
Description: Convert the app to a Progressive Web App (PWA) with a service worker that caches the home page,
crop info, and last scan result. Allow offline image capture and queue the analysis for when connectivity returns.
Backend Needed: Service Worker (Workbox); IndexedDB for offline queue; background sync API
#12 Admin Analytics Dashboard
P3 LOW
User Trigger: Admin logs in via a hidden /admin route
Description: Visualise total scans, top diseases detected, user registrations over time, and feature usage heatmaps.
Export to CSV. Helps the developer understand adoption and disease outbreak trends.
Backend Needed: Admin auth middleware; analytics aggregation queries (PostgreSQL); Chart.js / Recharts
dashboard; CSV export endpoint

■ Conclusion & Priority Action Plan
The Smart Crop Doctor project has a solid UI foundation and a clear vision. However, the current implementation is
not production-safe due to critical security vulnerabilities — primarily plain-text password storage and fully
client-side authentication. The mock AI analysis must also be replaced before the app can provide real value to
farmers.
Immediate Actions (Week 1–2): Fix B-01, B-02, B-03 (auth security). Set up PostgreSQL + JWT auth backend.
Remove the mock analysis and stub a real API endpoint.
Short Term (Month 1–2): Integrate a real ML model, add email OTP, implement scan history, add weather
integration. Fix all logical issues L-01 through L-05.
Medium Term (Month 3–6): Build community forum, PWA offline mode, voice assistant, soil calculator, and admin
dashboard.
